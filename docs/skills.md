## Skill: Add a New Encyclopedia Supplement

**Files involved:** `src/data/encyclopediaData.ts`

1. Open `encyclopediaData.ts`. Add a new entry to `encyclopediaSupplements[]` following the `EncyclopedialSupplement` interface exactly:
   ```ts
   {
     slug: 'magnesium-glycinate',          // URL slug — kebab-case
     name: 'Magnesium Glycinate',
     category: 'Sleep',                    // Performance | Sleep | Nootropics | Recovery | Health
     tagline: '...',
     evidenceTier: 'Moderate',             // Strong | Moderate | Emerging | Anecdotal
     primaryUse: '...',
     overview: '...',
     typicalDose: '200–400mg',
     bestFor: ['...', '...'],
     keyFacts: ['...', '...'],
     commonMistakes: ['...', '...'],
   }
   ```
2. The supplement will immediately appear in `EncyclopediaPage` — no route registration needed.
3. The deep dive for this slug will be auto-generated by Anthropic on first visit to `/deep-dive/magnesium-glycinate` and cached in Supabase.
4. To force-regenerate a cached deep dive: `POST /api/encyclopedia/deep-dive/:slug/refresh`.

## Skill: Add a New Backend API Route

**Files involved:** `server.js` or a new file under `api/`

1. Look at how `server.js` imports and mounts existing routes — match that pattern.
2. Create `api/your-feature.js`:
   ```js
   import { Router } from 'express';
   const router = Router();

   router.get('/', async (req, res) => {
     const supabase = getSupabaseService(); // import from server.js or pass via context
     const { data, error } = await supabase.from('products').select('*');
     if (error) return res.status(500).json({ success: false, error: error.message });
     res.json({ success: true, data });
   });

   export default router;
   ```
3. Register in `server.js`: `app.use('/api/your-feature', yourFeatureRouter);`
4. Apply `ingestLimiter` if the route mutates data; `generalLimiter` already covers `/api/*`.
5. Validate request body with Zod — import an existing schema from `src/schemas/` or create a new one.

**Files involved:** `src/pages/YourPage.tsx`, `src/components/SupplementAnalyzer.tsx`

1. Create `src/pages/YourPage.tsx`:

   ```tsx
     const { isDark } = useDarkMode();
     return (
       <div className="...tailwind classes...">
         {/* content */}
       </div>
     );
   }
   ```
2. In `SupplementAnalyzer.tsx`, import the component and add a `<Route>` inside the `<Routes>` block:
   ```tsx
   import YourPage from '../pages/YourPage';
   // ...
   <Route path="/your-path" element={<YourPage />} />
   ```
3. Add navigation to the page in the nav bar inside `SupplementAnalyzer.tsx` if needed.
4. Use Tailwind utility classes. Use `var(--css-variable)` for theme-aware colors (see `src/index.css` for variable definitions).

## Skill: Add a New Mobile Screen

**Files involved:** `SuppScannerApp/app/` (Expo Router)

### Tab screen (appears in bottom nav)
1. Create `SuppScannerApp/app/(tabs)/yourscreen.tsx`
2. Register it in `SuppScannerApp/app/(tabs)/_layout.tsx`:
   ```tsx
   <Tabs.Screen name="yourscreen" options={{ title: 'YourTitle' }} />
   ```
3. Add the tab to `BottomNav.tsx` in `SuppScannerApp/components/`.

### Standalone screen (pushed via router)
1. Create `SuppScannerApp/app/yourscreen.tsx` or `app/yourscreen/[param].tsx`
2. Navigate to it with `useRouter().push('/yourscreen')` or `router.replace('/yourscreen')`.

## Skill: Call the Backend API from Mobile

**Files involved:** `SuppScannerApp/src/config/api.js`, `SuppScannerApp/src/services/api.js`

`API_BASE_URL` is set from `EXPO_PUBLIC_API_URL` env var (`.env` in `SuppScannerApp/`). Defaults to `http://localhost:3001`.

**Simple fetch:**
```ts
import { API_BASE_URL } from '../../src/config/api';

const response = await fetch(`${API_BASE_URL}/api/your-endpoint`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ key: value }),
});
const result = await response.json();
```

**Via the supplementAPI wrapper** (for supplement-related calls):
```ts
import { supplementAPI } from '../src/services/api.js';
const result = await supplementAPI.getSupplement(barcode);
```

For production, set `EXPO_PUBLIC_API_URL=https://supplementscanner-production.up.railway.app` in the Expo environment.

## Skill: Work with Supabase (Backend)

**Files involved:** `src/services/dbService.js`, `scripts/supabase-schema.sql`

Always use `dbService.js` — don't instantiate a new Supabase client inline.

**Read a product:**
```js
import { DatabaseService } from './src/services/dbService.js';
const product = await DatabaseService.getProductByBarcode(barcode);
```

**Upsert a product:**
```js
await DatabaseService.upsertProduct(productData); // camelCase JS object → snake_case row
```

**Direct Supabase query (for new tables not in dbService):**
```js
// In server.js or an api/ route file:
const supabase = getSupabaseService();
const { data, error } = await supabase
  .from('your_table')
  .select('*')
  .eq('column', value);
```

**Adding a new table:**
1. Write the `CREATE TABLE` statement in `scripts/supabase-schema.sql`.
2. Run it in the Supabase SQL Editor (project `tdfveybkkikzltnhiawr`).
3. Add corresponding read/write methods to `dbService.js`.


## Skill: Add an Encyclopedia Deep Dive Endpoint Variation

The current deep dive endpoint is `GET /api/encyclopedia/deep-dive/:slug`. To add a variation (e.g., for premium RAG-based content):

1. Create a new route — **do not modify the existing endpoint**. Free and premium are separate pipelines.
2. Free path: `encyclopediaData.ts` slug → Anthropic prompt → JSONB cache in `supplement_deep_dives`
3. Premium path (not yet built): slug → vector retrieval (PubMed/NIH sources) → grounded Anthropic prompt (temperature 0.0) → source citations in response → separate Supabase table or `is_premium` flag
4. See `docs/context.md` → Business Model for full premium spec.
