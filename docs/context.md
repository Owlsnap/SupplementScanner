# context.md — Product & Domain Context

> For an AI agent to understand WHY things are built the way they are.

## Product Vision

SupplementScanner is two things built as one:

**1. Supplement Encyclopedia (web — primary product)**  
Browse and research 30+ supplements with evidence tiers (Strong / Moderate / Emerging / Anecdotal), typical doses, common mistakes, and AI-generated deep dives. The deep dive is the premium hook: a detailed clinical-style breakdown generated by Anthropic and cached in Supabase (30-day TTL).

**2. Barcode Scanner (mobile — companion)**  
Scan a supplement barcode with your phone camera → get an instant quality grade (A–F), ingredient breakdown, purity flags, and filler warnings. Calls the same backend API as the web app.

**Target market:** Swedish health-conscious consumers. The Swedish market is the first focus because supplement labeling in Sweden follows EU regulations, and the user has knowledge of local retailers (e.g., Tillskottsbolaget).

**Goal:** Be the trusted, no-nonsense supplement reference for Swedes — not a store, not a brand, just a tool.

## Domain Model

### Supplement (encyclopedia)
Defined in `src/data/encyclopediaData.ts`. Fields:
- `slug` — URL identifier (e.g., `creatine-monohydrate`)
- `name`, `category` (Performance / Sleep / Nootropics / Recovery / Health)
- `tagline`, `primaryUse`, `overview`
- `evidenceTier` — Strong / Moderate / Emerging / Anecdotal
- `typicalDose`, `bestFor[]`, `keyFacts[]`, `commonMistakes[]`

This is the **static layer** — hand-curated content for all 30 supplements.

### Deep Dive (AI-generated premium layer)
Stored in Supabase `supplement_deep_dives` table:
- `slug` (FK to encyclopedia entry)
- `content` (JSONB — Anthropic-generated detailed breakdown)
- `expires_at` (30-day TTL)

Generated on-demand by `GET /api/encyclopedia/deep-dive/:slug`. Cache-first: if Supabase has a non-expired row, return it; otherwise call Anthropic, store, return.

### Product (scanned supplement)
The barcode-scanned product record. Stored in Supabase `products` table. Key fields:
- `barcode`, `product_name`, `brand`, `category`, `sub_category`, `form`
- `ingredients` (JSONB array)
- `quality` (JSONB: `{ underDosed, overDosed, fillerRisk, bioavailability }`)
- `meta` (JSONB: `{ source, verified, lastUpdated }`)
- Type-specific data: `pre_workout_data`, `herb_data`, `protein_data` (JSONB, nullable)

### Scan (mobile flow)
User journey:
1. Home screen (`app/(tabs)/index.tsx`) — tap logo to open camera
2. `CameraView` detects barcode → `handleBarcodeScanned`
3. `POST /api/ingest/barcode/:code` — multi-layer extraction pipeline
4. If product needs manual fields → `manual-add.tsx`
5. Otherwise → `product/[barcode].tsx` — shows grade + ingredient breakdown
6. Result saved to `AsyncStorage` for scan history

### Quality Grade (mobile)
Computed client-side in `product/[barcode].tsx` from the `quality` object:
- Starts at 100, deducted for: underDosed (−30), overDosed (−20), fillerRisk high/moderate (−30/−15), bioavailability low/moderate (−20/−10)
- Maps to letter grade A/B/C/D/F

## Business Model

**Free tier:**
- Encyclopedia browsing (all 30 supplements)
- Barcode scanning + basic quality grade
- Ingredient list view

**Premium tier (planned, not yet built):**
- AI Deep Dive — evidence-grounded RAG analysis with PubMed/NIH sources
- Confidence scores weighted by study type (meta-analysis > RCT > animal study)
- "The Catch" — AI flags study limitations, dosage gaps between research and retail
- Interaction Engine — drug/supplement interactions (Red/Yellow/Green)
- Personalization — user health profile, dietary filters, timing tips
- Implementation: separate endpoint from free deep dive, RAG pipeline with vector store, Anthropic at temperature 0.0, source citations required in response

**Monetization also includes:** Affiliate links to Swedish retailers (if present in code — check `PremiumPage.tsx` and `MobileAppPage.tsx`).

**Shared subscription** with the encyclopedia site is planned — single account covers both.

## Swedish Market Specifics

- Product names and ingredient lists on scanned products may be in Swedish
- Retailer scraping targets Swedish-specific sites (see `src/extraction/siteExtractors/tillskottsbolagetExtractor.ts`)
- No i18n framework is in place — UI is currently in English
- Swedish regulatory context: supplements follow EU food supplement directive — ingredient labeling is mandatory, health claims are restricted
- If building new scrapers, check for Swedish-language selectors and UTF-8 characters (å, ä, ö)
